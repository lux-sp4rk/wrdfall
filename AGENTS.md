# AGENTS.md — Wordfall Agent Guide

> **TL;DR**: Godot 4.6 + React/Vite landing page. Main scene is `Boot.tscn`. Web export builds in CI. Supabase for auth/leaderboards.

## Project

Wordfall: Tetris-meets-Scrabble built with **Godot 4.6 (GDScript)**. Strategic word puzzles for iPad, mobile, and browser.

**Current state**: Feature-complete. Core game mode is **Loom Drop** (5×5 grid, letters fall, swipe to form words). Tutorial, stats, settings, pause, and high-score celebration are all live.

## Stack

| Layer | Technology | Key Files |
|-------|------------|-----------|
| Engine | Godot 4.6 | `godot/project.godot` → `scenes/Boot.tscn` |
| Scripts | GDScript (static typed) | `godot/scripts/` (~22 files) |
| Backend | Supabase | `godot/addons/supabase/`, `docs/supabase-auth-setup.md` |
| Landing | React + Vite | `landing/src/App.jsx` |
| Deploy | Vercel (landing + Godot web export) | `vercel.json`, `docs/deployment.md` |
| Tests | GUT (Godot), Vitest (landing) | `.github/workflows/test-coverage.yml` |

**Critical non-obvious facts:**
- **No Git LFS**. `dist/` and build artifacts are generated in CI; never commit them.
- **Web export is automated**: GitHub Actions builds Godot web exports for PRs and main pushes. You almost never need to manually export.
- **Deploy target is Vercel**, not Netlify. `dist/` is generated by `npm run build:all`.
- **Main scene is `Boot.tscn`**, not `Home.tscn`. Boot routes to Home (desktop) or LoomDrop (web).

## How to Work on This Project

### Run locally (web)
```bash
npm run serve        # Serves landing + Godot export locally
```

### Run Godot tests
```bash
cd godot && ./scripts/test.sh
```

### Run landing tests
```bash
cd landing && pnpm vitest run
```

### Open in Godot editor
Open `godot/project.godot` in Godot 4.6+. Main scene is `Boot.tscn`.

### Export for web (rarely needed)
1. Godot Editor → Project → Export → Web preset
2. Output goes to `landing/public/godot/` (auto-handled by `build:all`)

## Agent Workflow

1. **Always branch**: `git checkout -b feat/description` or `fix/issue-N-description`
2. **Never push to main**.
3. **Commit and push your branch when done** — even if it's WIP. CI will run tests.
4. **Open a PR** — Arachne (AI reviewer) and test-runner-faye run automatically.
5. **Update `docs/` if needed**: architecture changes → `ARCHITECTURE.md`, deploy changes → `deployment.md`.

### Git Protocol (MANDATORY)
When you make changes, you MUST complete the git workflow:
1. `git add <files>`
2. `git commit -m "type(scope): description"`
3. `git push origin <branch>`
4. Open PR via `gh pr create --title "..." --body "..."`

Never leave changes uncommitted. Never assume someone else will push.

### ACP Session Setup (MANDATORY for ACP/opencode sessions)
If you're running in an ACP/opencode session, the repo may not be cloned. You MUST:
1. Check if repo exists: `ls ~/Projects/wrdfall` or `ls /tmp/wrdfall`
2. If not, clone it: `git clone https://github.com/lux-sp4rk/wrdfall.git ~/Projects/wrdfall`
3. Always checkout the correct branch: `git checkout <branch-name>`
4. Verify you're on the right branch: `git branch --show-current`
5. Then proceed with changes and follow Git Protocol above

Never work on `main` directly. Never make changes without checking the branch first.

## Common Tasks

| Task | Where to Start |
|------|----------------|
| Fix game logic bug | `godot/scripts/LoomDrop.gd` |
| Add new game mechanic | `LoomDrop.gd` + `GameConstants.gd` + `FeatureFlags.gd` |
| UI / theme change | `ThemeManager.gd`, `ThemeConstants.gd`, or scene `.tscn` files |
| Add screen/menu | `godot/scenes/` + matching `.gd` script |
| Landing page change | `landing/src/App.jsx` or `landing/src/screens/` |
| Auth / leaderboard | `godot/addons/supabase/`, `docs/supabase-auth-setup.md` |
| Word validation | `godot/scripts/Dictionary.gd` + `godot/scripts/LanguageConfig.gd` |
| CI / build issue | `.github/workflows/`, `docs/deployment.md` |

## Key Patterns

- **Autoloads (Singletons)**: `GameSettings`, `StatsManager`, `ThemeManager`, `GameConstants`, `FeatureFlags`, `Supabase`. Access anywhere: `GameSettings.some_setting`
- **FeatureFlags**: Use `FeatureFlags.is_enabled("flag_name")` to gate unfinished features.
- **Theme system**: Colors and fonts live in `ThemeConstants.gd`. UI updates propagate via `ThemeManager.theme_changed` signal.
- **Persistence**: `GameSettings.save_settings()` writes to user config. `StatsManager` handles session data and high scores.
- **Word validation**: `Dictionary.gd` validates words against a compressed dictionary. `LanguageConfig.gd` per-language settings.
- **Web bridge**: `Boot.gd` detects web vs desktop and routes accordingly.

## Gotchas & Warnings

- `dist/` is in `.gitignore`. Do not commit build output.
- The `.env` for Supabase (`godot/addons/supabase/.env`) is gitignored. Publishable keys only.
- **Row Level Security (RLS) is mandatory** on Supabase tables. See `docs/deployment.md`.
- Godot web exports can be slow to load on mobile. Texture compression is enabled (`fix/mobile-wasm-load-issue-152`).
- The `bug-hunter-d33` workflow runs automated bug detection on PRs. It can be noisy but catches real issues.
- If you see `encountered-file-should-be-pointer-but-wasnt` — that's an old LFS ghost. It's been fixed; ignore it.
